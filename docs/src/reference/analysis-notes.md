# Reference notes

The `docs/analysis/` directory contains rebuild homework and parity design notes.

These files are intentionally kept in the repository for maintainers and deep protocol debugging. For day-to-day integration, start with the Guide pages first.

## When to read analysis docs

- You need to understand parity gaps in detail
- You are debugging protocol-level behavior
- You are contributing to core command generation or image pipeline logic

## Recommendation

Use this docs site for onboarding and usage. Use `docs/analysis/` as deep reference material when you need implementation context.
